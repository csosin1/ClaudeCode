# Gym Intelligence — Basic-Fit Competitive Tracker

A Python tool that tracks the competitive gym landscape across Basic-Fit's core European markets (Netherlands, Belgium, France, Spain, Luxembourg, Germany) using OpenStreetMap data, AI-powered chain classification, and quarterly competitive analysis.

**Source code:** [github.com/csosin1/ClaudeCode/tree/claude/gym-intelligence-tool-Onwxp/gym-intelligence](https://github.com/csosin1/ClaudeCode/tree/claude/gym-intelligence-tool-Onwxp/gym-intelligence)

## Tech Stack

- **Python 3.11+**
- **SQLite** — single-file portable database (`gyms.db`)
- **Streamlit** — mobile-friendly web dashboard
- **Plotly** — interactive charts and scatter maps
- **Anthropic Python SDK** — Claude-powered classification and analysis (claude-sonnet-4-20250514)
- **OSM Overpass API** — free gym location data (no API key needed)
- **APScheduler** — quarterly automated pipeline runs

## Setup

```bash
cd gym-intelligence

# Install dependencies
pip install -r requirements.txt

# Initialize the database
python db.py

# Set your Anthropic API key (required for classification and analysis)
export ANTHROPIC_API_KEY=your-key-here
```

## Usage

### First Run — Collect Data

```bash
# Collect gym locations from OpenStreetMap (takes 10-30 minutes)
python collect.py

# Classify chains using Claude (requires ANTHROPIC_API_KEY)
python classify.py

# Generate quarterly analysis
python analyze.py
```

### Or run the full pipeline at once

```bash
python scheduler.py --now
```

### Launch the Web UI

```bash
streamlit run app.py
```

The app will be available at `http://localhost:8501`.

### Scheduled Runs

```bash
# Start the scheduler (runs on first Monday of each quarter)
python scheduler.py
```

## Database Schema

| Table | Description |
|-------|-------------|
| `locations` | Individual gym locations with OSM data, coordinates, address fields, and chain linkage |
| `chains` | Canonical chain names with AI classification, price tier, 18-month normalized cost, and membership model |
| `snapshots` | Point-in-time location counts per country per chain (for QoQ comparison) |
| `quarterly_analyses` | Full markdown competitive analysis reports generated by Claude |

## Data Pipeline

1. **Collect** (`collect.py`) — Queries OSM Overpass API for `leisure=fitness_centre`, `leisure=sports_centre`, and `amenity=gym` across 6 country bounding boxes. Upserts locations, normalizes chain names via fuzzy matching, writes snapshots.

2. **Classify** (`classify.py`) — Sends unclassified or stale chains to Claude for competitive classification (direct competitor / non-competitor / unknown), price tier estimation, and 18-month cost normalization. For chains with 10+ locations, fetches website pricing pages for more accurate extraction.

3. **Analyze** (`analyze.py`) — Generates a quarterly markdown report covering market structure, chains on the rise/decline, new entrants, country narratives, and an overall threat assessment for Basic-Fit.

## Web UI Pages

1. **Market Overview** — Country-filtered market share bar charts among direct competitors, with chain detail cards showing QoQ changes, price tier, and 18-month cost
2. **Chain Explorer** — Individual chain profiles with classification rationale, location-over-time line charts, Plotly scatter_mapbox of all active locations, and QoQ growth metrics by country
3. **Competitive Analysis** — Browse and read AI-generated quarterly reports with metadata
4. **Admin / Refresh** — Trigger the full pipeline with live progress, manually review/override chain classifications, export all data to CSV
